# Costing Master 网页设计规格

> 版本: v1.0 | 日期: 2026-05-26 | 状态: 已确认

## 一、产品定义

为"造价大师 Costing Master" Skill 构建一个面向专业用户的展示型网站。目标不是展示 AI 能力，而是让造价人员快速判断：能否帮自己处理项目文件、输出什么、边界在哪。

### 不是什么
- 不是可运行的后端服务
- 不是聊天机器人界面
- 不是造价软件替代品

### 是什么
- 一个纯静态 SPA，展示产品价值、工作流、示例成果和信任机制
- 零依赖部署：单个 index.html 文件，双击即可在浏览器中打开

---

## 二、技术方案

| 维度 | 决策 |
|------|------|
| 部署形式 | 纯静态单文件 `index.html` |
| 路由 | Hash 路由 (`#home`, `#workflow/settlement` 等) |
| CSS | 全手写，不依赖 Tailwind/Bootstrap |
| JS | 原生 JavaScript，无框架 |
| 字体 | Google Fonts: Inter (正文) + JetBrains Mono (数字/表格) |
| 外部依赖 | 零 |

### 文件结构
```
costing-master-web/
└── index.html   ← 唯一的部署文件
    ├── <style>  — 全局CSS
    ├── <script> — SPA路由 + Tab切换 + 组件交互
    └── 所有页面模板 (通过Hash切换显示/隐藏)
```

## 三、视觉风格：专业工具风

### 设计理念
类似 Bloomberg Terminal、Wind 金融终端——严肃、克制、以表格和数据为核心。避免渐变、大圆角、阴影等消费级 SaaS 风格。

## 五、首页设计

### 第一屏：Hero + 角色入口
- 深色背景 (`#1a1d23`)
- 左右分栏：左为主标题 + CTA 按钮，右为四类角色卡片
- 主标题：**"把合同、清单、结算书整理成可复核的造价审核底稿。"**
- 副标题包含四级判断标签图例
- 两个主按钮："查看结算审核示例" + "上传文件试用"
- 四类角色卡片：造价员/预算员、结算员/审核人员、商务经理/成本负责人、招投标人员

### 第二屏：信任条
- 白色背景，三栏均分
- 不编造 / 可溯源 / 会自检

### 第三屏：三步流程
- 浅灰背景 (`#fafafa`)
- "三个步骤，从资料到工作底稿"
- 步骤1：上传与识别 → 步骤2：补充与抽取 → 步骤3：复核与导出
- 每个步骤使用带编号圆圈 + 简短描述

### 第四屏：底部 CTA
- 深色背景
- "选择一个身份开始，或直接查看示例"
- 两个按钮引导至示例中心

## 六、工作流详情页设计

每个工作流页包含三部分：

### 页面头部
- 工作流名称 + 描述
- 输入/输出/依据标签条
- 浅灰背景

### 三步向导面板
- 步骤指示器（连接线 + 圆圈，已完成/当前/待解锁三种状态）
- 步骤1 内容区：文件拖拽上传区（虚线边框）+ 文件类型提示卡片
- 步骤2/3 在静态版中以示例数据预填充展示

### 示例输出区
- 四级判断标签图例
- 数据表格（标题行灰色背景，数字列右对齐 + 等宽字体，判断列使用彩色标签）
- 每条数据行有确定/疑似/待补/人工确认的标签

---

## 七、示例中心设计

### 页面结构
- 左侧：流程时间线（上传 → 缺资料提示 → 抽取字段 → 生成台账 → 二次审核 → 下载）
- 右侧：输出成果 Tab 切换（审减台账 / 争议事项 / 需补资料 / 管理层摘要）
- 顶部：背景信息条（项目类型、所在地、合同类型、适用标准）
- 使用该 Skill 的真实示例文案填充

---

## 八、信任机制页面

### 双模式说明页
- 左右对比卡片：快速参考模式(免费) vs 项目工作模式(专业版)
- 清晰列出每种模式"能做什么"和"不能做什么"
- 专业版卡片使用橙色边框强调

### 二次审核机制页
- 标题："不是 AI 说了算——每条结论都经过二次审核"
- 10项质检清单（2列5行网格）
- 三级结论说明（通过/需修正/禁止交付，带颜色区分）

### 安全与边界页
- 明确列出：不出具法律意见、不替代签章、不替代审计结论
- 不编造数据、缺资料时拒绝给确定金额
- 数据隐私和保密说明

## 九、核心组件规范

### 按钮
- 主按钮：`#e36209` 背景，白色文字，`border-radius: 4px`
- 深色背景下次按钮：透明背景 + `#30363d` 边框 + `#c9d1d9` 文字

### 判断标签
- 四个等级各有独立的背景色 + 边框色 + 文字色
- 尺寸：`padding: 2px 6px; font-size: 10px; border-radius: 3px`

### 数据表格
- 表头：`#f5f5f5` 背景，左对齐文字列，右对齐数字列
- 数据行：`1px solid #f0f0f0` 底部边框
- 数字列使用 `JetBrains Mono` 等宽字体
- 无金额时显示灰色 `—`

### 步骤指示器
- 已完成：`#e36209` 橙色圆圈 + 连接线
- 当前：`#e36209` 圆圈，下方文字橙色
- 待解锁：`#e0e0e0` 灰色圆圈，文字 `#999`

### 导航栏
- 深色背景 `#1a1d23`
- 品牌名左侧，导航项右侧
- 当前页：白色文字 + 橙色下划线
- 其他：`#8b949e` 灰色文字

## 十、内容规范

### 术语要求
所有页面文案必须对齐 GB/T 50500-2024 术语：
- 使用"最高投标限价"，不使用"招标控制价"
- 使用"安全生产措施费"，不使用"安全文明施工费"
- 使用"增值税"，不使用"税金"
- 使用"工程量清单缺陷"，不使用"清单错误"

### 金额格式
- 千分位逗号分隔
- 两位小数
- 标注"元"
- 审减金额带负号前缀

### 免责声明
每份示例输出的末尾必须展示免责声明：
> "本输出为工作底稿，不替代注册造价工程师签章、企业审批或最终结算定案文件。"

## 十一、开发范围

### MVP (当前阶段)
所有 16 个页面，静态展示，模拟数据填充。

### 不在范围内
- 实际文件解析 (Excel/PDF/Word)
- 后端 API
- AI Agent 集成
- 用户认证
- 数据持久化

---

## 十二、验收标准

1. 所有 Hash 路由可正常切换，浏览器前进/后退正常
2. 首页 Hero、信任条、三步流程、CTA 完整展示
3. 7 个工作流页各有完整的头部 + 三步向导 + 示例输出
4. 示例中心包含结算审核的完整流程时间线和 Tab 切换
5. 数据表格中的数字使用等宽字体，金额格式正确
6. 四级判断标签颜色方案一致
7. 双模式说明和二次审核页面内容完整
8. 移动端响应式：768px 以下可正常阅读（非优先但不可崩塌）
9. 零外部依赖：禁用 CDN JS/CSS 库
10. 术语对齐 GB/T 50500-2024
